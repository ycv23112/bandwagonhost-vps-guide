## 官网访问的几种常见情况

**情况一：主域名打不开，备用地址可用**

换成 bwh81.net 试试。这是官方认可的镜像域名，购买和管理功能完全一样。

**情况二：两个地址都打不开**

大概率是本地网络或 DNS 的问题。可以试这几步：

1. 换一个 DNS 服务器（比如 8.8.8.8 或 114.114.114.114）
2. 清除浏览器缓存和 DNS 缓存（Windows 下 `ipconfig /flushdns`，Mac 下 `sudo dscacheutil -flushcache`）
3. 换个网络环境试试（手机 4G 热点切换进来）
4. 如果以上都不行，大概率是运营商 DNS 污染，用梯子访问就可以

**情况三：能打开但是加载很慢**

正常现象。搬瓦工官网服务器在海外，国内访问天然有延迟，不影响使用。

## VPS 的 IP 被封：比官网打不开更麻烦的问题

很多人买了搬瓦工之后遇到的不是官网问题，而是 VPS 本身的 IP 被封了，服务器连不上。这两件事容易混淆，说清楚：

- **官网打不开** = 你登录搬瓦工后台的界面进不去，但你的 VPS 本身不受影响
- **VPS 的 IP 被封** = 你的 VPS 服务器的出口 IP 被 GFW 屏蔽，从国内 ping 不通，SSH 连不上

后者影响更直接，VPS 完全用不了。

### 怎么判断 IP 是不是被封了

最简单的方法是用搬瓦工 KiwiVM 后台内置的检测工具：

1. 登录搬瓦工账户，进入 **Services → My Services**
2. 找到对应的 VPS，点击进入 **KiwiVM Control Panel**
3. 在浏览器地址栏里把 URL 末尾的 `main.php` 改成 `main-exec.php?mode=blacklistcheck`，回车
4. 点击 **Test Main IP** 按钮，等几秒，看结果

如果结果显示 IP 在黑名单里，就是被封了。

也可以从国内多个节点 ping 一下你的 VPS IP——如果国内全部超时但国外 ping 得通，基本可以确认是被 GFW 屏蔽，而不是服务器本身的问题。

### IP 为什么会被封

常见原因有这几个：

- 用 VPS 搭建了国内 GFW 策略敏感的服务，IP 被触发屏蔽
- IP 段被批量封锁（不一定是你自己的操作问题，可能是同段 IP 里其他用户被封了，波及到你）
- 长时间高频发包被识别为异常流量

碰到这种情况，换 IP 或换机房是最直接的解法。如果频繁被封，可以考虑换线路质量更好的机房（比如香港 CN2 GIA），不同机房的 IP 段稳定性差异比较大。
